# 第三阶段：媒体层复盘

日期：2026-06-12
阶段：媒体层首个实现切片

## 结论

第三阶段首个切片达到了目标：个人站不再只是写作和作品入口，而有了照片、音乐和轻量阅读互动。当前不应急着关闭第三阶段，因为真实音频、照片素材质量、灯箱焦点管理和素材 attribution 仍需要打磨。

## 已完成

- `/photos`：精选照片、照片网格、灯箱。
- `/music`：当前歌单、曲目列表、迷你播放器。
- 媒体数据模型：`Photo`、`Track`、`Mix`。
- 首页媒体入口接入 `/music`。
- 博客文章页新增 reader mode、鼠标跟随阅读微光、代码块行数。
- Vercel production 已上线。

## 做对的地方

- 媒体层没有做成普通相册模板。
- 音乐没有伪装成完整音乐应用，先保留为工作室状态层。
- 博客互动保持克制，服务阅读，不把宠物或强装饰塞进正文。
- 移动端无横向溢出，照片和音乐页面第一屏成立。

## 仍需补充

- 真实照片素材质量和 attribution 策略。
- 是否引入真实音频文件。
- 灯箱焦点管理和移动端手势。
- 播放器是否要抽象为可复用的 `NowPlaying` 组件。

## 不建议补进第三阶段的内容

- 全站搜索：属于第四阶段。
- 复杂命令菜单：属于第四阶段。
- 相册后台、上传、CMS：属于第五阶段。
- 宠物组件：更适合 `/lab`，不适合直接进入博客正文。

## 第四阶段进入条件

第四阶段可以开始，但不应该把第三阶段视为完全关闭。当前更合理的方式是并行推进：

- 第四阶段先做全站命令入口和搜索。
- 第三阶段后续继续补真实素材、焦点和媒体质量。
