## 同步协议

1. 先更新本地 Markdown。
2. 本地检查和提交。
3. 推送到 GitHub。
4. 使用 `docs +update --command overwrite` 更新飞书文档。
5. 抓取飞书文档并验证关键文本。
6. 如果节点或 doc token 变化，更新本文件。

## 命令模板

```bash
lark-cli docs +update \
  --api-version v2 \
  --as user \
  --doc '<doc_token>' \
  --command overwrite \
  --doc-format markdown \
  --content @docs/PROJECT_MAP.md
```
