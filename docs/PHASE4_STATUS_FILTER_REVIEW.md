# 第四阶段状态面板与筛选复盘

日期：2026-06-13

## 结论

`StatusPanel + FilterBar` 切片已经完成，并且方向成立：它把第四阶段从“只有 Command Center 的交互层”推进到“页面表面也有状态反馈和范围控制”。

但这个切片还没有证明 `FilterBar` 的跨场景复用能力。它已经服务 `/blog` 和 `/projects`，下一步必须进入 `/photos`，因为照片页有真实标签、视觉网格、灯箱和移动端压力，更能验证这个组件是否足够稳。

## 已执行内容

- 首页新增 `StatusPanel`：最近在写、最近在做、最近在听。
- 新增可复用 `FilterBar`。
- `/blog` 使用新版筛选条。
- `/projects` 使用新版筛选条。
- 筛选条显示结果数量、active 状态和显性清除动作。
- 增加筛选空状态壳。
- 飞书和 GitHub 追踪已同步。

## 做对的地方

- 状态面板没有变成 dashboard，只保留三类当前状态。
- 每张状态卡都链接到真实页面，没有做不可操作的装饰状态。
- 筛选条开始承担结果反馈，而不是只是一组 pill button。
- 清除动作显性可见，避免用户遗忘当前范围。

## 暴露的问题

- `/photos` 仍没有复用 `FilterBar`，组件还没有通过媒体场景验证。
- `PROJECT_MAP.md` 的“下一步”段落仍写着已经完成的 `StatusPanel` 和 `FilterBar`，需要修正。
- `GlobalSearch` 是否要独立抽象仍未被证明；当前内容量不足以支持立刻拆分。
- 第三阶段媒体层虽然已首版完成，但照片筛选和灯箱联动还没有正式记录。

## 下一步进入标准

进入“照片筛选与媒体浏览联动”切片：

- `/photos` 复用 `FilterBar`。
- 支持 `Featured` 和照片标签筛选。
- 筛选后结果数量准确更新。
- 筛选后精选区域和网格都跟随当前结果。
- 灯箱左右导航只在当前筛选结果内移动。
- 移动端仍无页面级横向溢出。
- 暂不做多选筛选和 URL query 同步。
