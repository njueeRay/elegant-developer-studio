# 第四阶段照片筛选复盘

日期：2026-06-13

## 结论

照片筛选切片成立。`FilterBar` 已经通过三类场景验证：

- `/blog`：文字列表。
- `/projects`：项目卡片网格。
- `/photos`：媒体网格、精选区域、灯箱联动。

这说明它可以进入后续 `/knowledge`，但还不需要扩展成多选 faceted navigation。当前更重要的是建立 `Knowledge` 的内容模型和信息气味。

## 已执行内容

- `/photos` 复用 `FilterBar`。
- 支持 `All`、`Featured` 和照片 tags。
- 精选区、照片网格、灯箱导航都跟随当前筛选结果。
- 切换筛选时关闭灯箱，避免 active photo 和结果集合不一致。
- 更新项目地图、路线图、设计系统、QA、版本追溯和飞书知识库。
- GitHub issue `#3` 和 `#5` 已追加追溯评论。

## 做对的地方

- 没有引入复杂筛选抽屉，保持媒体页低负担。
- `Music` 这类单结果筛选仍能保持精选区、网格和灯箱一致。
- 保留显性 `Clear`，用户不会被困在筛选状态。
- 移动端只允许筛选条内部横向滚动，页面本身无横向溢出。

## 暴露的问题

- `/knowledge` 仍是未来入口，还没有公开可访问索引。
- Command Center 里 `Knowledge is planned` 已经开始拖慢 IA 演进。
- `FilterBar` 是否需要支持多选仍缺乏真实压力测试。
- 公开知识库需要可引用机制，否则很难在 GitHub、飞书和文章之间流动。

## 下一步进入标准

进入第五阶段首个切片：`Knowledge` 信息架构与筛选模型。

进入范围：

- 新增 `/knowledge` 公开页面。
- 新增知识内容模型。
- 支持按知识类型筛选。
- 每条知识有稳定锚点和复制引用动作。
- Command Center 将 `knowledge` 从 planned suggestion 改成真实结果。
- 暂不做详情页、全文搜索、多选 facets 和大型图谱。
