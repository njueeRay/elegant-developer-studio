## 使用方式

1. 看方向：先读项目地图。
2. 看为什么：读 PRD、信息架构、决策记录。
3. 看当前进度：读开发进度和版本追溯。
4. 看质量：读质量验收。
5. 提需求或评论：优先评论对应飞书页面；下一轮开发会同步回 GitHub issue 和仓库文档。

## 同步规则

- 仓库文档是事实源。
- 飞书是协作层。
- 每个阶段完成后必须同步：
  - `PROJECT_MAP.md`
  - `PROGRESS_LOG.md`
  - `VERSION_TRACE.md`
  - 对应阶段页面或 issue
- 长期决策必须写入 `DECISIONS.md`。
